# Ошибки и улучшения

## Обнаруженные ошибки и несоответствия
- Тесты ожидают методы `enable_wifi` и `disable_wifi`, но в `busylib/client.py` их нет. Текущие тесты упадут.
- README показывает использование `version_info.version`, тогда как `VersionInfo` содержит только `api_semver` и клиент возвращает именно его. Документация расходится с кодом.
- `VersionInfo` ограничен одним полем; если API возвращает доп. данные (version, branch), они теряются и вызывают неожиданное поведение при строгом парсинге.
- Нет тайм-аутов/ретраев для HTTP-запросов: зависания при сетевых проблемах.
- Отсутствует клиентская валидация входных данных (диапазоны яркости/громкости, валидность пути/SSID и т.п.); ошибки всплывут только с сервера.
- APIMismatch: `/api/assets/upload` (POST) присутствует в `openapi.yaml`. Ранее полученный 404 был вызван использованием GET. Необходимо проверить поддержку DELETE и реальную доступность POST (текущие проверки затруднены нестабильностью устройства).
- Покрытие тестами: нет проверок для BLE-методов, `get_screen_frame`, `delete_app_assets`; негативные сценарии проверены только для версии.
- Плейсхолдеры в Makefile (`lint`, `test-cov`, `docs`) не реализованы.

## Возможные улучшения
- Добавить отсутствующие Wi-Fi методы или скорректировать тесты/README для согласованности API.
- Пополнить модели (например, `VersionInfo`) или явно фильтровать лишние поля от сервера.
- Ввести тайм-ауты по умолчанию и (опционально) стратегию повторов.
- Расширить тесты: BLE, скриншоты, удаление ассетов, негативные ветки для большинства методов.
- Выделить валидацию входных данных (диапазоны, обязательные поля) до отправки запросов.
- Убрать/реализовать Makefile-плейсхолдеры; добавить линтер/coverage цели.

## Переход на Pydantic (обоснование)
- Централизованная валидация схем: можно описать строгие типы, диапазоны и обязательные поля; ранние ошибки на клиенте.
- Удобная сериализация/десериализация: `model.model_dump()` убирает Enum/датаклассы вручную, избавляет от `_serialize_for_json`.
- Работа с лишними полями: `model_validate` может игнорировать/строго проверять неожиданные ключи, снижая риск немых несоответствий API.
- Богатые ошибки с локализацией поля (в тестах и логах понятнее, что не так).

## Переход на HTTPX (обоснование)
- Современный клиент с sync/async интерфейсом: можно дать пользователю выбор и улучшить производительность при множестве запросов.
- Тайм-ауты как составной объект (connect/read/write/pool), удобная настройка и безопасные дефолты.
- Поддержка HTTP/2, connection pooling и прокси из коробки.
- Встроенные ретраи через транспорт или внешние стратегии; проще расширять политику отказоустойчивости.
- Совместимость с asyncio-стеком и контекстными менеджерами; тесты можно писать на `httpx.MockTransport` без `requests-mock`.
