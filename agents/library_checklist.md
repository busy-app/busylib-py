# Чеклист качества библиотеки (busylib, 2025)

- [x] Современный HTTP-клиент с тайм-аутами и retry — httpx, дефолтные тайм-ауты и простой backoff защищают от зависаний.
- [x] Строгие модели и валидация — Pydantic v2, enum’ы, диапазоны; ранние ошибки и предсказуемые структуры.
- [x] Корректная работа с Unicode/JSON — `ensure_ascii=False`, charset в заголовке; русский текст уходит как UTF-8.
- [ ] Логирование — сейчас нет; нужно debug-логи запросов/ответов и warning для ошибок для быстрой диагностики.
- [~] Ретраи/политика ошибок — есть повторы на transport errors, но без экспоненциального backoff/джиттера и фильтрации по кодам.
- [x] Тестируемость — MockTransport в тестах, покрытие основных путей; можно расширить негативные/edge-кейсы.
- [ ] Документация — README и примеры не обновлены под новый клиент; нужны актуальные usage и заметки о валидации/тайм-ауте.
- [x] Структура пакета — src layout, `package-dir` настроен, публичный surface в `__init__.py`.
- [ ] Метаданные/дистрибуция — обновить версию/описание под новую API, добавить лицензии в classifiers; проверить long_description.
- [~] Типизация — аннотации есть, но нет `py.typed`; стоит добавить для mypy/pyright.
- [ ] Исключения/сообщения — API/Request ошибки без контекста (URL, метод); добавить для дебага.
- [x] Асинхронность — есть AsyncBusyBar на httpx.AsyncClient для полной поддержки asyncio.
- [~] Конфигурируемость — базовые параметры (timeout/retries/backoff) есть, но нет прокси/HTTP2/кастомных заголовков.
- [ ] Юзабилити/UX — нет CLI/quickstart, helper’ов для типовых операций; добавить примеры и удобные обёртки.
